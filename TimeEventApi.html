post /timeEvents
Adds a new timecard event (addTimecardEvent)
By passing in the appropriate options, you can create a new timecard event, e.g. clock in, clock out, etc. Employee id is not necessary, since the employee should be authenticated and have a session

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

timecardEvent TimeEvent (required)
Body Parameter — TimecardEvent item to add

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

event created

400

invalid input, object invalid

409

an existing item already exists
Up 
delete /timeEvents
Deletes an existing timecard event (deleteTimecardEvent)
Deletes a timecard event by eventKey

Query parameters

Query Parameter — the key for the time event that will be deleted

Responses

400

Up 
get /timeEvents
searches time events (searchTimeEvent)
By passingin the appropriate parameters, you can search for a time event

Query parameters

employeeKey (optional)
Query Parameter — search events by employee key
date (optional)
Query Parameter — search events by date format: date-time
payPeriod (optional)
Query Parameter — search events by pay period key

Return type

array[TimeEvent]

Example data

Content-Type: application/json
[ {
  "eventKey" : 1,
  "approve" : "Y",
  "action" : 2.0,
  "employeeId" : 42,
  "time" : "2019-08-29T09:12:33.001Z"
}, {
  "eventKey" : 1,
  "approve" : "Y",
  "action" : 2.0,
  "employeeId" : 42,
  "time" : "2019-08-29T09:12:33.001Z"
} ]

put /timeEvents
Updates an existing timecard event (updateTimcardEvent)
By passing in the appropriate options, you can update an existing timecard event. This allows you to update the time, approve the event, or change the action code

Consumes

  • application/json

Request body

timecardEvent TimeEvent (required)
Body Parameter — TimecardEvent item to update

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses
